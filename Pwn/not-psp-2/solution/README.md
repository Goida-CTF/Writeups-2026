# Решение
Учимся билдить блобы для arm9 через arm-none-eabi-as:
```
arm-none-eabi-as -o blob.o blob.S
arm-none-eabi-objcopy -O binary blob.o blob.bin
```
Собираем (вайбкодим?) persistent mode харнесс для afl++, который будет грузить наши блобы поверх другого рома и запускать только arm9. Не забываем включить JIT. Фаззим минут 10-20 при 50 exec/sec.

Ищем интересные краши, находим один краш (или зависание) с джампом ведущим посередине кода. Находим источник - место с `printf("miauz")`.

Аккуратно собираем на месте, куда прыгает неправильный джамп, шеллкод в константах (которые JIT инлайнит). Не забываем, что место джампа и вообще возникновение бага зависит от контекста в блоке, контекст придется подобрать (кодекс помогает найти в коде релевантные детали).

Шеллкод должен добиться чтения флага и записи в screenshot.png, а потом выйти с кодом 0.
